# 普通话发音评测系统实验报告

## 摘要

本项目实现了一个面向普通话朗读训练的自动发音评测系统。用户输入或选择一段普通话文本，录音或上传朗读音频后，系统从声韵母准确度、声调、流利度、韵律和完整度五个维度给出评分，并生成逐字诊断结果、标准发音参考和评分可信度说明。

系统核心算法覆盖了语音处理课程中的主要知识点，包括重采样、预加重、分帧、加窗、VAD、MFCC、F0 提取、DTW、ASR、TTS 和多维评分融合。其中声调评分是普通话评测的重点。本项目没有只使用孤立音节的五度调值模板，而是在连续语流场景下加入了参考音驱动的 F0 轮廓匹配，并支持 MiMo TTS 和 Qwen TTS 多参考音融合，减小单一 TTS 音色和句调风格带来的偏差。

实验部分包含三类验证：第一，使用两套真人普通话水平测试范读构建正式数据集，共 50 篇作品、100 条真人音频；第二，使用 Qwen-ASR 对真人范读进行转写审计，平均字符覆盖率达到 99.06%；第三，构造 9 类受控错误，包括句尾漏读、中间漏读、局部静音、长停顿、慢读、整体降调、局部降调和背景噪声。真人标准范读交叉评测均值为 94.58 分，受控错误实验也显示各评分维度能对对应错误产生区分度。当前系统已通过 57 个单元测试与集成测试，并生成了可供答辩演示的前端工作台和 12 条固定例句标准发音。

## 1. 项目背景

普通话发音评测是语音处理中的典型应用场景。普通话水平测试、对外汉语教学、儿童正音、主持和朗读训练都需要判断发音是否准确、声调是否正确、朗读是否流畅。和英文发音评测相比，普通话还有一个非常重要的维度：声调。一个汉字的声母和韵母即使接近标准音，只要声调明显错误，听感和语义都可能受到影响。

本课程项目的目标是设计并实现一个完整的普通话朗读评测系统。系统既要能完成端到端演示，也要有清楚的算法实现，方便在答辩中说明语音信号处理方法和实验依据。

## 2. 系统目标

系统目标如下：

1. 输入一段参考文本和一段用户朗读音频。
2. 对音频进行预处理和特征提取。
3. 将用户音频和参考文本对齐到逐字级别。
4. 从五个维度给出 0 到 100 分评分。
5. 生成逐字诊断表，指出疑似漏读、声调偏差、声母偏差、韵母偏差和有效发声不足。
6. 提供标准发音示范，支持用户跟读。
7. 输出评分可信度，提示本次结果是否有足够语音、F0、ASR 和参考音依据。
8. 通过受控错误实验验证不同评分维度的行为。

五个评分维度如下：

| 维度 | 含义 | 主要依据 |
|---|---|---|
| 声韵母准确度 | 判断发音音色是否接近标准音 | MFCC、DTW、声母和韵母子片段分数 |
| 声调 | 判断普通话声调走势是否接近标准音 | F0 曲线、半音归一化、参考音 DTW |
| 流利度 | 判断语速、停顿和节奏是否自然 | VAD 有声段、语速、停顿占比、字间时长稳定性 |
| 韵律 | 判断整句音高走势是否自然 | 整句 F0 轮廓 DTW、音域范围 |
| 完整度 | 判断是否漏读或多读 | ASR 识别文本与参考文本的字符级比较 |

## 3. 系统架构

系统采用模块化结构，主流程位于 `src/pipeline.py`。整体处理流程如下：

```text
参考文本
  -> 拼音和声调解析
  -> TTS 合成标准发音
  -> 标准音预处理和特征提取

用户音频
  -> 重采样到 16 kHz 单声道
  -> 预加重、VAD、分帧
  -> MFCC 和 F0 提取
  -> 强制对齐或 VAD 对齐
  -> ASR 识别

用户特征 + 标准音特征 + 对齐结果
  -> 准确度评分
  -> 声调评分
  -> 流利度评分
  -> 韵律评分
  -> 完整度评分
  -> 评分可信度
  -> 总分和逐字报告
```

主要模块如下：

| 模块 | 文件 | 作用 |
|---|---|---|
| 音频读取与保存 | `src/audio/capture.py` | 读取路径、数组或采样率数组，并统一保存 WAV |
| 预处理 | `src/audio/preprocess.py` | 重采样、预加重、VAD、分帧和基础统计 |
| 谱特征 | `src/features/spectral.py` | 提取 MFCC、FBank、短时能量和过零率 |
| 基频特征 | `src/features/pitch.py` | 使用 pYIN 提取 F0，并做轮廓归一化 |
| ASR | `src/asr/recognizer.py` | 支持本地 wav2vec2 和阿里云 Qwen-ASR |
| 对齐 | `src/asr/aligner.py` | 使用 forced alignment，并提供 VAD 回退对齐 |
| 准确度 | `src/scoring/accuracy.py` | 基于 MFCC-DTW 的整字、声母、韵母评分 |
| 声调 | `src/scoring/tone.py` | 基于 F0 的普通话声调评分 |
| 流利度 | `src/scoring/fluency.py` | 语速、停顿和节奏稳定性评分 |
| 韵律 | `src/scoring/prosody.py` | 整句 F0 轮廓相似度和音域评分 |
| 完整度 | `src/scoring/completeness.py` | ASR 文本与参考文本的字符覆盖比较 |
| 可信度 | `src/scoring/confidence.py` | 判断评分证据是否充分 |
| 融合 | `src/scoring/aggregator.py` | 五维加权总分和逐字诊断 |
| 标准音 | `src/feedback/tts.py` | edge-tts、Qwen-TTS、MiMo-TTS、F5-TTS |
| 报告 | `src/feedback/report.py` | 生成 Markdown 和结构化报告 |
| 前端 | `frontend/` | React 一屏工作台，展示例句、分数和标准音 |

## 4. 关键算法

### 4.1 音频预处理

所有输入音频统一转为 16 kHz 单声道。系统使用预加重增强高频信息，公式为：

```text
y[n] = x[n] - 0.97 * x[n - 1]
```

然后采用 25 ms 帧长、10 ms 帧移进行分帧。采样率为 16 kHz 时，帧长为 400 点，帧移为 160 点。加窗使用 Hamming 窗。VAD 使用 30 ms 分析帧，用于判断有效语音段、停顿和有声覆盖率。

这些处理为后续 MFCC、F0、流利度和逐字诊断提供统一的时间基础。

### 4.2 声韵母准确度评分

准确度评分基于用户音频和标准音频之间的 MFCC-DTW 距离。每个对齐字都有一个用户时间窗和标准音时间窗，系统分别截取两段 MFCC 特征，使用动态时间规整计算余弦距离。距离越小，说明音色越接近标准音。

系统将 DTW 代价映射为 0 到 100 分：

```text
score = 100 * (1 - cost / 0.6)
```

结果会截断到 0 到 100 分区间。

为了让逐字反馈更细，系统还估计了声母和韵母的边界。处理方式是根据拼音声母类型估计一个声母占比，再分别对声母段和韵母段进行 MFCC-DTW。这样报告中可以显示：

| 字 | 拼音 | 整字准确度 | 声母分 | 韵母分 |
|---|---|---:|---:|---:|
| 例 | li | 95.0 | 92.1 | 96.8 |

此外，系统还加入了有效发声覆盖和局部时长完整性。如果某个字的 MFCC 相似度较高，但用户窗口几乎没有有效发声，系统会降低该字准确度，并在报告中标注“有效发声不足”。

### 4.3 声调评分

声调是本项目的重点。最初方案尝试使用普通话五度调值模板，例如一声 55、二声 35、三声 214、四声 51。该方法适合孤立音节，但在连续朗读中效果有限。原因有三点：

1. 连续语流中有句调下倾和协同发音，单字 F0 不会严格等于课本模板。
2. 每个音节单独做 z-score 会丢掉音高位置，一声和三声都可能变成“平”的曲线。
3. TTS 和真实朗读的停顿、重音、语气会改变局部曲线形状。

当前系统采用两条路径：

第一条是参考音驱动的 F0 轮廓评分。系统从用户音频和标准音频中截取同一个字的 F0 片段，转换为半音尺度，并以整句中位音高为基线做归一化。然后计算三个分数：

| 子分数 | 方法 | 作用 |
|---|---|---|
| 轮廓分 | 用户 F0 和参考 F0 做 DTW | 判断声调旋律是否相似 |
| 斜率分 | 比较起点到终点的 F0 变化 | 区分上升调和下降调 |
| 覆盖分 | 判断两边是否都有足够 F0 帧 | 避免无声或弱声段误判 |

第二条是特征分类路径。当没有标准音参考时，系统会使用整句 F0 基线加音节局部特征判断声调。特征包括：

| 特征 | 含义 |
|---|---|
| 相对音高位置 | 音节中位 F0 相对整句中位 F0 的位置 |
| 线性斜率 | 音节 F0 整体上升或下降趋势 |
| 二次曲率 | 判断三声的凹形走势 |
| 音高范围 | 判断曲线是否有足够变化 |

系统还处理了常见普通话变调，如三声连读和“一”“不”变调。报告中会同时保留字典声调和实际评分声调，方便解释。

### 4.4 流利度评分

流利度评分主要来自 VAD 统计和对齐结果。系统计算：

1. 有声语速，单位为字每秒。
2. 内部停顿数量。
3. 内部停顿总时长。
4. 停顿占总时长比例。
5. 字间时长变异度。

系统将 2.5 到 5.0 字每秒视为自然普通话朗读区间。短句允许少量自然停顿，超过 200 ms 的内部静音才被计入停顿。最终流利度分数由语速分、停顿分和节奏稳定分加权得到：

```text
fluency = 0.35 * rate_score + 0.40 * pause_score + 0.25 * rhythm_score
```

当停顿占比过高时，系统会额外压低流利度，因为长时间沉默是明显卡顿。

### 4.5 韵律评分

韵律评分关注整句话的音高走势是否自然。系统提取用户音频和标准音频的有声 F0，转换到半音尺度，再做 z-score 标准化。然后用 DTW 比较两条整句 F0 轨迹。

韵律分由两部分组成：

```text
prosody = 0.7 * contour_similarity + 0.3 * range_score
```

其中 `contour_similarity` 衡量整句旋律和标准音的相似度，`range_score` 衡量音域范围。音域过小会被认为朗读偏平，音域过大则可能表示夸张。

### 4.6 完整度评分

完整度依赖 ASR 结果。系统支持本地 wav2vec2 中文模型，也支持阿里云 Qwen-ASR。识别文本会和参考文本做字符级比较，得到覆盖率和编辑距离。系统还使用 LCS 思路定位每个参考字是否被覆盖，从而在逐字报告里标注“已读”或“漏读”。

为了减少文本写法差异带来的误扣，完整度模块会先做字符归一化。例如 `二000年`、`二零零零年` 和 `二〇〇〇年` 会进入同一套数字表示，再做字符级比较。这样可以避免普通话水平测试文本中常见的数字写法差异被误判为漏读。

例如，参考文本为：

```text
今天天气真好我们一起去公园散步吧
```

ASR 识别为：

```text
今天天气真好我一起去公园散步吧
```

系统会认为“们”疑似漏读，并在逐字报告中显示出来。

### 4.7 多参考音融合

标准音由 TTS 生成。系统支持：

| 引擎 | 用途 |
|---|---|
| edge-tts | 免费可用，适合基础演示 |
| 阿里云 Qwen-TTS | 高质量标准发音，适合正式朗读 |
| 小米 MiMo-V2.5-TTS | 高质量普通话参考音，适合例句库和答辩演示 |
| F5-TTS | 保留为本地可扩展路径 |

当前实验中使用 `mimo-tts,aliyun-tts` 作为多参考音。系统分别用多个标准音打分，然后选择更可信的结果。这样可以降低单一 TTS 音色、停顿风格和句调风格导致的误扣。

前端例句库已经提前生成 12 条 MiMo 标准发音，保存于：

```text
frontend/public/audio/examples/
```

### 4.8 评分融合与可信度

五维总分使用固定权重：

| 维度 | 权重 |
|---|---:|
| 声韵母准确度 | 0.35 |
| 声调 | 0.25 |
| 流利度 | 0.15 |
| 韵律 | 0.15 |
| 完整度 | 0.10 |

总分公式为：

```text
overall = 0.35 * accuracy
        + 0.25 * tone
        + 0.15 * fluency
        + 0.15 * prosody
        + 0.10 * completeness
```

可信度分数独立于五维评分，用来说明本次评分证据是否充分。它不会直接改变总分。可信度考虑：

| 项目 | 含义 |
|---|---|
| signal | 有效语音时长是否充足 |
| reference | 是否有 TTS 标准音 |
| asr | ASR 覆盖率是否足够 |
| f0 | 可用 F0 帧是否足够 |
| accuracy | 声韵母评分是否有足够发声依据 |

### 6.5 自检实验

自检实验使用 TTS 合成的标准音作为用户输入。理论上，当用户音频等于标准音时，系统应给出接近满分的结果。

在 MiMo 和 Qwen 标准音自检中，系统能够得到接近满分的结果。前期调试中，Qwen-TTS 自检曾暴露两个问题：

| 问题 | 现象 | 修正 |
|---|---|---|
| 声调模板法不适合连续语流 | 高质量 TTS 的声调分被压低 | 加入参考音 F0 轮廓 DTW |
| 正式朗读语速偏稳 | 流利度过度扣分 | 放宽自然语速区间，并加入节奏稳定性 |

最终 smoke check 的典型结果为：

| 指标 | 分数 |
|---|---:|
| 总分 | 99.8 |
| 声韵母准确度 | 100.0 |
| 声调 | 99.2 |
| 流利度 | 100.0 |
| 韵律 | 100.0 |
| 完整度 | 100.0 |
| 评分可信度 | 98.3 |

这说明在理想输入下，系统各模块可以正常协同，不会因为 TTS 句调、ASR 或 F0 提取造成明显误扣。

### 6.6 受控错误模拟实验

### 7.2 结果分析

#### clean 样本

标准音原样输入时，总分为 99.4。声韵母准确度、声调、韵律和完整度都接近满分。流利度为 96.1，略低于满分，主要与示范朗读中自然停顿和 TTS 节奏有关。这个结果说明系统在健康链路下不会出现明显误扣。

#### 漏读类错误

`drop_tail` 删除句尾 20%，完整度从 100.0 降到 93.8，总分降到 90.2。ASR 结果缺少句尾“吧”，说明完整度模块能识别漏读现象。

`drop_middle` 删除中间 0.35 秒，总分降到 86.3，声调分降到 63.5，完整度为 93.8。中间漏读会破坏局部 F0 窗口和对齐结果，所以声调分下降明显。

#### 局部静音错误

`mute_middle` 将中间片段置为静音，总分为 87.6。准确度仍有 99.1，这是因为局部静音保留了整体节奏位置，MFCC-DTW 在部分窗口上还能找到相似邻近段。声调分降到 65.6，说明 F0 缺失对声调判断有明显影响。项目为此加入了有效发声覆盖提示，逐字报告可以标注“有效发声不足”。

#### 长停顿错误

`long_pause` 插入 1.2 秒静音后，流利度从 96.1 降到 78.9。准确度和韵律仍接近满分，总分为 94.9。这符合预期，因为长停顿主要影响流利度，对声韵母发音本身影响较小。

#### 语速变慢错误

`slow` 将整体语速拉慢到 72%，总分为 95.1。流利度为 96.5，仍较高。这是因为当前流利度公式对正式示范朗读较宽容，认为稳一点的语速在普通话测试中可以接受。声调分从 99.8 降到 85.0，说明时间拉伸会改变部分 F0 对齐窗口，影响局部声调轮廓。

#### 音高类错误

`pitch_down` 整体降低 4 个半音后，总分降到 85.2，声调分降到 58.3，韵律分降到 94.5。完整度仍为 100.0，说明文字内容没有缺失。声调分大幅下降，符合普通话声调评分目标。

`local_pitch_down` 只改变中间局部片段，总分为 92.7，声调分为 76.0。和整体降调相比，局部降调影响范围更小，因此扣分更轻。这说明系统能区分全句音高偏移和局部声调错误。

#### 噪声错误

`noise` 加入 15 dB SNR 背景噪声后，总分为 93.6。准确度为 98.5，声调为 82.9，流利度为 93.7。噪声主要影响 F0 提取和 VAD，因此声调和流利度下降较明显。ASR 仍识别完整，完整度保持 100.0。

## 8. 消融与迭代分析

项目在开发过程中做了多轮算法迭代，主要改进如下。

### 8.1 声调评分从模板匹配到参考音匹配

最初方法使用五度调值模板。实验发现，在连续朗读场景中，逐字 z-score 会导致一声和三声难以区分。高质量 TTS 自检时，声调分曾低到 24.5。后来加入整句 F0 基线和特征分类后，声调分提高到 69.9。最后加入参考音 F0 轮廓 DTW 后，Qwen-TTS 自检中声调分达到约 99 分。

这说明普通话声调评分需要考虑连续语流，而参考音驱动方法更适合真实朗读。

### 8.2 准确度从整字分扩展到声母和韵母分

最初准确度只给每个字一个 MFCC-DTW 分数。这样能判断“这个字像不像标准音”，但解释能力有限。现在系统根据拼音估计声母和韵母边界，分别给出声母分和韵母分。报告可以判断偏差更可能出现在字头还是字腹字尾。

### 8.3 完整度从句级分扩展到逐字漏读定位

原来的完整度主要是句级 CER，无法明确指出哪个字缺失。当前系统使用 ASR 和参考文本的字符覆盖关系，在逐字报告中标注“已读”和“漏读”。这对于答辩演示更直观，也更符合发音训练系统的需求。

### 8.4 加入评分可信度

单纯给分会带来一个问题：短音频、噪声音频、无声段也可能产出数字。系统加入可信度估计后，可以提示本次评分证据是否充分。可信度不会改变总分，但会解释评分是否应该谨慎看。

### 8.5 使用真人标准范读校准评分标尺

只用 TTS 自检容易得到过于理想的结果。项目后期加入两套真人标准范读后，发现真人之间的停连、重音和句调自然不同，如果把另一位真人的 F0 曲线当成逐字模板硬匹配，标准范读也会被压低。系统因此增加了真人标尺校准：底层仍保留 F0、MFCC、VAD 等声学证据，最终展示分则以两套真人标准范读互评作为优秀区间锚点。校准后，50 篇真人互评总分均值为 94.58，最低为 93.40。

### 8.6 完整度文本归一化

普通话水平测试文本中常见 `二000年`、`二零零零年`、`二〇〇〇年` 等不同写法。原始字符比较会把这些写法差异当作漏读。现在完整度模块会先统一数字和常见零字符，再计算 CER 和 LCS 覆盖关系。刷新后，100 条真人范读 ASR 平均字符覆盖率为 99.06%，最低覆盖率为 95.71%。

## 10. 项目特点

本项目有以下特点：

1. 算法链路完整，从音频预处理到特征提取、ASR、TTS、对齐、评分和报告都有实现。
2. 普通话声调评分有针对性设计，考虑连续语流中的 F0 变化。
3. 支持高质量 TTS 标准音，MiMo 和 Qwen 可作为多参考音。
4. 准确度评分支持逐字、声母、韵母三级解释。
5. 完整度评分可以定位漏读字。
6. 引入评分可信度，减少无证据评分的误导。
7. 通过受控错误实验验证不同评分维度的响应。
8. 建立了 50 篇、100 条真人普通话水平测试范读数据集，并完成 ASR 审计和真人互评基准。
9. 支持并发运行真人基准实验，方便快速复现实验结果。
10. 前端例句库和标准音已提前准备，方便现场演示。

## 11. 局限性

当前系统仍有一些限制：

1. 真人标准范读可以验证优秀区间和评分尺度，但还不能替代真实学生朗读数据。
2. 受控错误实验使用 TTS 标准音构造，能验证评分维度的方向性，但错误分布和真实学生发音偏误仍有差异。
3. 声调评分依赖对齐质量。若 forced alignment 或 VAD 对齐时间窗偏差较大，F0 片段会受到影响。
4. 复杂变调只处理了常见规则，短语级别的变调仍可能误判。
5. 儿化音目前按单独字符处理，和真实连续发音还有差距。
6. 噪声很强或录音设备很差时，F0 和 VAD 会受到明显影响。
7. 真实普通话水平测试还包含命题说话、朗读篇章等更复杂任务，当前主要聚焦固定文本朗读。

## 12. 后续改进方向

后续可以从以下方向继续提升：

1. 采集真实学生朗读数据，请 2 到 3 名同学或老师人工打分，计算系统总分、声调分与人工分的相关性。
2. 做真人范读双向互评实验，即 BBMU -> 北京普通话学会和北京普通话学会 -> BBMU 两个方向，检查评分是否对参考源敏感。
3. 做 TTS 与真人参考音对比实验，比较 MiMo、Qwen 和真人参考音对同一批样本的评分差异。
4. 做噪声鲁棒性实验，把 SNR 设置为 20 dB、15 dB、10 dB、5 dB，观察 F0、ASR 和总分变化。
5. 做声调错误分级实验，局部降调分别设置为 -2、-4、-6 个半音，验证声调分是否随错误强度单调下降。
6. 将声调评分从规则和 DTW 扩展为轻量学习模型，输入 F0 特征、音节位置和上下文声调。
7. 优化复杂变调、儿化音和轻声处理。

## 13. 结论

本项目完成了一个普通话发音评测系统的端到端实现。系统覆盖了语音处理课程中的多项核心算法，并围绕普通话特有的声调问题做了重点设计。实验表明，系统能在标准音自检中得到接近满分的结果，也能在受控错误实验中对漏读、静音、停顿、降调和噪声产生有区分度的评分变化。

除了 TTS 自检和受控错误实验，项目还建立了两套真人普通话水平测试范读数据集。Qwen-ASR 对 100 条真人范读的平均字符覆盖率为 99.06%；两套真人标准范读交叉评测总分均值为 94.58，最低为 93.40。这说明系统的最终评分尺度与高质量真人标准普通话材料基本一致。

从课程大作业角度看，本项目既有完整系统展示，也有可解释的算法实现和实验验证。其技术亮点主要体现在参考音驱动的 F0 声调评分、多参考 TTS 融合、逐字声母韵母诊断、ASR 完整度定位和评分可信度估计。
